üìò About Exit Codes

AutoSys Workload Automation reports exit codes to indicate the success or failure of jobs. A code of 0 typically means success. Non-zero or negative codes point to errors, system issues, or agent problems.

üö´ Internal AutoSys Exit Codes

  • -655: Max restart attempts exceeded
  • -656: Job not yet started or still running
  • -657: Agent lost track of job processes; use chase -E -A for verification

üß™ Common Exit Codes & Fixes

  • 0: Successful execution (Windows & Unix/Linux)

  • 7: Storage control blocks destroyed (Windows error)

  • 12: Low memory/swap space or ulimit -u exceeded
  • 15: Job terminated (often due to server reboot)
  • 28: No space left on device ‚Äî check disk and inode usage
  • 126: Command is not executable or has permission issue

  • 128: Invalid exit argument (must be 0‚Äì255)

üîê Security-Related Exit Code -655 (AD Binding)

Error -655 can occur due to:

  • Account not in autosys_secure table
  • AD account locked, disabled, or restricted
  • Password mismatch between AD and autosys_secure
  • Account lacks "log on as service/batch" rights on Windows
